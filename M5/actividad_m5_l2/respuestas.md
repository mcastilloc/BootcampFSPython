
### ¿Qué es un modelo de datos y para qué sirve en bases relacionales?

Un **modelo de datos** es una representación conceptual y lógica de cómo se organizan, almacenan y relacionan los datos dentro de una base de datos. Define las **entidades**, sus **atributos** y las **relaciones** entre ellas.

En bases de datos relacionales, el modelo de datos sirve para:
- Diseñar la estructura de las tablas.
- Definir relaciones entre tablas.
- Evitar duplicidad de información.
- Garantizar la integridad y coherencia de los datos.
- Facilitar el mantenimiento y la escalabilidad del sistema.

### ¿Qué es una clave foránea y qué garantiza?

Una **clave foránea (Foreign Key)** es un campo o conjunto de campos en una tabla que hace referencia a la **clave primaria** de otra tabla.

La clave foránea garantiza:
- **Integridad referencial**, asegurando que los datos relacionados existan.
- Consistencia entre tablas relacionadas.
- Que no se creen registros con referencias inválidas.
- Control sobre acciones como borrado o actualización (por ejemplo, borrado en cascada).
